# UI/UX Guidelines VPN Panel

## Назначение

Документ определяет единые правила проектирования пользовательского интерфейса VPN Panel.

Он является основой для работы дизайнеров и разработчиков Frontend.

Любые изменения дизайн-системы должны сначала отражаться в данном документе.

---

# Основные принципы

Интерфейс должен быть:

- современным;
- минималистичным;
- понятным;
- быстрым;
- адаптивным;
- доступным.

Главная цель — уменьшить количество действий пользователя для выполнения любой операции.

---

# Design First

Все новые экраны сначала создаются в Figma.

После утверждения дизайн реализуется в React.

Запрещается создавать новые экраны без предварительного проектирования.

---

# Design System

Проект использует единую дизайн-систему.

Она включает:

- цвета;
- типографику;
- отступы;
- размеры;
- компоненты;
- состояния элементов;
- иконки;
- анимации.

Все новые компоненты должны соответствовать этой системе.

---

# Цветовая палитра

Используются семантические цвета.

## Primary

Основной цвет интерфейса.

Используется для:

- основных кнопок;
- активных элементов;
- ссылок.

---

## Secondary

Второстепенные действия.

---

## Success

Используется для:

- успешных операций;
- активных сервисов;
- зелёных индикаторов.

---

## Warning

Используется для:

- предупреждений;
- истечения срока действия;
- необходимости внимания пользователя.

---

## Error

Используется для:

- ошибок;
- удаления;
- критических событий.

---

## Info

Используется для информационных сообщений.

---

# Типографика

Используется единая система размеров.

Например:

- H1
- H2
- H3
- Body
- Small
- Caption

Запрещается использовать произвольные размеры текста.

---

# Отступы

Используется единая шкала.

Например:

```
4
8
12
16
24
32
48
64
```

Все компоненты должны использовать эту шкалу.

---

# Скругления

Используется единый набор радиусов.

Например:

```
sm

md

lg

xl
```

---

# Тени

Используется ограниченный набор уровней.

Например:

```
none

sm

md

lg
```

---

# Иконки

Используется только Lucide React.

Запрещается смешивать несколько библиотек иконок.

---

# Компоненты

Основой UI являются компоненты shadcn/ui.

Допускается создание собственных компонентов только при отсутствии подходящего стандартного решения.

---

# Повторное использование

Любой повторяющийся элемент интерфейса должен быть вынесен в отдельный компонент.

Дублирование JSX не допускается.

---

# Формы

Все формы должны иметь:

- подписи полей;
- валидацию;
- сообщения об ошибках;
- индикатор загрузки;
- блокировку повторной отправки.

---

# Таблицы

Таблицы должны поддерживать:

- сортировку;
- фильтрацию;
- поиск;
- пагинацию;
- адаптивное отображение.

---

# Кнопки

Каждая кнопка должна иметь понятное действие.

Не рекомендуется использовать несколько одинаково выделенных действий на одном экране.

---

# Диалоги

Для подтверждения опасных операций используются модальные окна.

Например:

- удаление;
- сброс;
- отключение.

---

# Уведомления

Используются Toast-уведомления.

Типы:

- Success
- Warning
- Error
- Info

---

# Загрузка данных

Во время загрузки отображаются:

- Skeleton;
- Spinner;
- Progress Bar.

Не допускаются пустые экраны без индикации загрузки.

---

# Пустые состояния

Если данных нет, отображается понятное сообщение и возможное действие пользователя.

---

# Адаптивность

Интерфейс должен корректно работать на:

- Desktop;
- Laptop;
- Tablet.

Поддержка мобильных устройств может быть реализована отдельным этапом развития проекта.

---

# Навигация

Навигация должна быть последовательной и предсказуемой.

Пользователь всегда должен понимать:

- где он находится;
- куда может перейти;
- как вернуться назад.

---

# Доступность (Accessibility)

Следует учитывать:

- контрастность;
- навигацию с клавиатуры;
- видимый фокус;
- альтернативные подписи;
- корректную семантическую разметку.

---

# Производительность

Следует минимизировать:

- лишние перерисовки;
- тяжёлые анимации;
- загрузку неиспользуемых компонентов.

Использовать ленивую загрузку там, где это оправдано.

---

# Консистентность

Одинаковые действия должны выглядеть одинаково во всём приложении.

Пользователь не должен переучиваться при переходе между разделами.

---

# Запрещается

Запрещается:

- использовать разные стили для одинаковых элементов;
- создавать новые цвета без необходимости;
- смешивать несколько дизайн-систем;
- нарушать принципы адаптивности;
- использовать нестандартные компоненты без архитектурного обоснования.

---

# Заключение

Интерфейс VPN Panel должен быть современным, единообразным и масштабируемым.

Все новые экраны и компоненты должны соответствовать данным правилам.
