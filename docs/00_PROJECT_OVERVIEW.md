# VPN Panel

## О проекте

VPN Panel — современная система управления VPN-инфраструктурой.

Проект разрабатывается как единая экосистема, состоящая из нескольких независимых сервисов, взаимодействующих через API.

Основная задача проекта — предоставить безопасную, масштабируемую и удобную платформу для управления VPN.

Проект ориентирован на дальнейшее развитие без изменения базовой архитектуры.

---

# Основные цели

Проект должен обеспечить:

- управление VPN-серверами;
- управление VPN-клиентами;
- управление пользователями;
- мониторинг состояния инфраструктуры;
- централизованное администрирование;
- безопасную работу через REST API;
- возможность масштабирования.

---

# Архитектурная философия

Проект строится вокруг принципа разделения ответственности.

Каждый компонент отвечает только за собственную область.

```
                Browser
                    │
                    │
            React Frontend
                    │
             REST API (HTTPS)
                    │
             FastAPI Backend
                    │
        ┌───────────┴───────────┐
        │                       │
 PostgreSQL               HostAgent API
                                │
                    VPN Server / Docker
```

Backend не взаимодействует напрямую с VPN.

Все операции выполняются через HostAgent.

---

# Основные компоненты

## Frontend

Назначение:

- пользовательский интерфейс;
- отображение данных;
- работа с REST API;
- визуализация статистики.

Frontend не содержит бизнес-логики.

---

## Backend

Назначение:

- конфигурация приложения;
- авторизация;
- бизнес-логика;
- управление пользователями;
- управление VPN;
- работа с БД;
- взаимодействие с HostAgent.

Backend является центральным компонентом системы.

---

## HostAgent

HostAgent представляет собой отдельный сервис.

Он устанавливается непосредственно на VPN-сервер.

Он отвечает за:

- запуск VPN;
- остановку VPN;
- чтение конфигураций;
- выполнение системных команд;
- получение статистики;
- управление Docker;
- взаимодействие с WireGuard / AmneziaWG / Xray.

HostAgent никогда не предоставляет пользовательский интерфейс.

---

## PostgreSQL

Используется как единая база данных проекта.

Backend является единственным компонентом, имеющим доступ к БД.

---

# Принципы архитектуры

Проект придерживается следующих принципов.

## Single Responsibility

Каждый модуль отвечает только за одну задачу.

---

## API First

Все взаимодействия выполняются через API.

Даже внутренние сервисы взаимодействуют одинаковым способом.

---

## Async First

Backend полностью асинхронный.

---

## Stateless Backend

Backend не хранит состояние между запросами.

---

## Масштабируемость

Любой компонент можно заменить или масштабировать независимо.
