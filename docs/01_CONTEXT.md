# Контекст проекта VPN Panel

## Назначение документа

Этот документ фиксирует все согласованные решения и правила разработки проекта.

Он является частью официальной документации проекта и должен обновляться при каждом изменении архитектуры, процессов или принятых решений.

---

# Цель проекта

Создать современную веб-панель управления VPN-инфраструктурой.

Система должна быть:

- безопасной;
- масштабируемой;
- расширяемой;
- независимой от конкретного VPN-решения;
- удобной для сопровождения.

---

# Основные компоненты

Проект состоит из независимых сервисов:

- Frontend
- Backend
- HostAgent
- PostgreSQL
- Infrastructure

Каждый компонент имеет собственную область ответственности.

# Архитектурные принципы

В проекте используются следующие принципы:

- API First
- Async First
- Single Responsibility
- Separation of Concerns
- Stateless Backend

---

# Правила принятия решений

Перед изменением архитектуры необходимо:

1. Проанализировать существующее решение.
2. Обсудить изменение.
3. Принять решение.
4. Обновить документацию.
5. После этого изменять код.

---

# Правила разработки

При разработке соблюдаются следующие правила:

- небольшие логические изменения;
- один завершённый этап = один коммит;
- не менять архитектуру без согласования;
- не выполнять рефакторинг без необходимости;
- сначала анализ существующего кода, затем изменения;
- сначала документация, затем реализация.

---

# Документация

Каталог `docs/` является официальным источником информации о проекте.

Все архитектурные изменения должны отражаться в документации.

Документация и код должны оставаться синхронизированными.

# Долгосрочная цель

Построить систему, которую можно развивать без изменения фундаментальной архитектуры.

Каждый новый модуль должен интегрироваться в существующую структуру без нарушения уже принятых решений.
