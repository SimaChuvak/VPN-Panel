# Architecture Decision Records (ADR)

## Назначение

Документ содержит журнал архитектурных решений (Architecture Decision Records).

Каждое значимое решение фиксируется отдельно и содержит:

- дату;
- статус;
- описание проблемы;
- принятое решение;
- последствия.

Изменение ранее принятых решений допускается только через новую запись.

---

# ADR-0001

## Название

Выбор архитектуры Backend

### Статус

Accepted

### Дата

2026-07

### Контекст

Необходимо выбрать архитектуру, которая позволит масштабировать проект и поддерживать его в долгосрочной перспективе.

### Решение

Использовать многослойную архитектуру:

```
API
↓
Services
↓
Repositories
↓
Database
```

### Последствия

Положительные:

- высокая поддерживаемость;
- тестируемость;
- слабая связанность компонентов.

Отрицательные:

- больше файлов;
- выше порог входа.

# ADR-0004

## Название

Разделение Backend и HostAgent

### Статус

Accepted

### Контекст

Необходимо безопасно управлять VPN-инфраструктурой.

### Решение

HostAgent работает как отдельный сервис.

Backend обращается к HostAgent через API.

Frontend не взаимодействует с HostAgent напрямую.

### Последствия

Положительные:

- безопасность;
- масштабируемость;
- возможность подключения нескольких HostAgent.

# ADR-0010

## Название

Проектирование до реализации

### Статус

Accepted

### Решение

Перед реализацией новой функциональности должны быть подготовлены:

- архитектурное описание;
- документация (при необходимости);
- дизайн (для UI);
- план реализации.

Только после этого начинается разработка.
